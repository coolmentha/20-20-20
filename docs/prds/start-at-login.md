# 开机自启

## 问题陈述

用户希望 20-20-20 护眼提醒应用可以在 Windows 登录后自动启动，这样不需要每天手动打开应用，也能持续获得护眼提醒和站立活动提醒。

开机自启会改变系统登录后的应用行为，必须由用户主动控制。应用通过开机自启启动时不应该打扰用户，应该保持提醒能力可用并静默进入托盘。

## 解决方案

新增一个 Windows 第一版开机自启功能。用户在托盘菜单中通过“开机自启”勾选项开启或关闭，操作即时同步到操作系统登录项。

应用以操作系统登录项状态作为事实来源，不额外维护一份应用内开机自启配置。普通手动启动应用时继续显示主窗口；只有通过开机自启参数启动时，应用才托盘启动。托盘启动后计时器立即开始累计主动看屏时间。

## 用户故事

1. 作为电脑使用者，我希望可以在托盘菜单里开启开机自启，以便登录 Windows 后无需手动打开应用。
2. 作为电脑使用者，我希望可以在托盘菜单里关闭开机自启，以便应用不再随系统登录自动启动。
3. 作为电脑使用者，我希望开机自启开关即时生效，以便不需要重启应用或额外保存。
4. 作为电脑使用者，我希望应用随系统登录启动后静默进入托盘，以便不被主窗口打断。
5. 作为电脑使用者，我希望开机自启后提醒计时立即开始，以便护眼提醒和站立活动提醒保持有效。
6. 作为电脑使用者，我希望手动打开应用时仍然显示主窗口，以便主动查看当前倒计时和休息状态。
7. 作为电脑使用者，我希望退出应用不改变开机自启设置，以便退出当前进程和关闭自启是两个独立操作。
8. 作为电脑使用者，我希望当系统不支持或无法读写开机自启状态时，应用不要显示虚假的成功状态。

## 实现决策

- 第一版只保证 Windows 开机自启。
- 入口只放在托盘菜单中，使用“开机自启”勾选项；不在主窗口新增设置区。
- 以操作系统登录项状态作为事实来源，不新增应用内开机自启配置文件。
- 用户勾选或取消勾选后，立即同步到操作系统登录项。
- 开发模式不注册真实 Windows 开机自启。
- 登录项启动时携带可识别启动参数。只有通过该参数启动时，应用才托盘启动。
- 普通手动启动应用时继续显示主窗口。
- 托盘启动后，计时器立即开始累计主动看屏时间。
- 退出应用不改变开机自启设置。
- 读取或写入登录项失败时，托盘菜单保留“开机自启（不可用）”禁用项，并保持旧状态。

## 测试决策

- 验证托盘菜单能显示当前操作系统登录项状态。
- 验证勾选“开机自启”会即时写入 Windows 登录项。
- 验证取消勾选“开机自启”会即时移除 Windows 登录项。
- 验证开发模式不会注册真实 Windows 开机自启。
- 验证通过开机自启参数启动时，主窗口不主动显示，应用进入托盘并继续计时。
- 验证普通手动启动时，主窗口仍然显示。
- 验证退出应用不会改变登录项状态。
- 验证登录项状态读取或写入失败时，菜单项显示为禁用的“开机自启（不可用）”，不会显示虚假成功。
- 实现后运行现有自动化测试和项目构建。

## 范围外

- macOS 开机自启。
- Linux 开机自启。
- 主窗口设置页。
- 应用内配置文件持久化开机自启状态。
- 安装器、发布流水线或正式制品上传。
- 开机后自动显示主窗口。
- 改变现有护眼提醒、站立活动提醒、暂停、隐藏到托盘或退出语义。

## 补充说明

- 领域语言应遵循现有词汇表：开机自启、托盘启动、主动看屏时间。
- 现有应用已经具备托盘入口和隐藏到托盘能力，本功能应复用托盘交互，不扩大主窗口尺寸。
- 上一版“每小时站立活动提醒”明确把启动行为排除在范围外，因此本功能作为独立 PRD 处理。
