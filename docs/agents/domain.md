# 领域文档

本文件说明工程类技能在探索代码前如何读取本仓库的领域文档。

## 读取顺序

- 先读取仓库根目录的 `CONTEXT.md`。
- 如果未来出现根目录 `CONTEXT-MAP.md`，则按其中映射读取相关上下文。
- 如果存在 `docs/adr/`，读取与当前任务相关的 ADR。
- 如果相关文件不存在，静默继续；不要因为缺少 ADR 或上下文文件而主动创建。

## 文档布局

本仓库采用 single-context 布局：

```text
/
├── CONTEXT.md
├── docs/
│   └── adr/
└── src/
```

`CONTEXT.md` 是领域词汇表入口。`docs/adr/` 用于记录重要架构决策，当前可为空。

## 术语规则

输出 issue、PRD、测试名、重构建议或诊断结论时，优先使用 `CONTEXT.md` 已定义的领域术语。

如果需要使用的概念不在词汇表中，先判断是否真的需要新增概念。若确实需要，后续通过 `grill-with-docs` 补齐术语或 ADR。

## ADR 冲突

如果输出内容与现有 ADR 冲突，必须显式说明冲突点，而不是静默覆盖既有决策。
